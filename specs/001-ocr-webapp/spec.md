# Feature Specification: OCR Web Application

**Feature Branch**: `001-ocr-webapp`
**Created**: 2026-03-29
**Status**: Draft
**Input**: User description: "Python + Streamlit + OpenCV + Gemini Flash を使った OCR Webアプリ"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - カメラで撮影して文字を抽出する (Priority: P1)

ユーザーはブラウザ上でカメラを起動してその場で撮影し、撮影した画像内の文字テキストを自動で抽出して画面上に表示される。

**Why this priority**: OCR アプリの中核機能であり、これなしにアプリは成立しない。

**Independent Test**: カメラを起動して撮影し、撮影した画像の文字がテキストとして表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** ユーザーがアプリを開いている、**When** カメラを起動して撮影する、**Then** 撮影した画像内の文字がテキストとして表示される
2. **Given** ユーザーが撮影した、**When** 処理が実行される、**Then** 処理中であることを示す状態がユーザーに伝わる
3. **Given** カメラへのアクセスが拒否されている、**When** カメラを起動しようとする、**Then** 権限が必要である旨のメッセージが表示される
4. **Given** カメラを搭載していないデバイスの場合、**When** アプリを開く、**Then** カメラが利用できない旨のメッセージが表示される

---

### User Story 2 - ログインして認証済みユーザーのみアクセスできる (Priority: P2)

アプリへのアクセスはログインフォームで保護されており、正しいユーザー名とパスワードを入力したユーザーのみ OCR 機能を利用できる。API の不正利用による課金を防ぐことが目的。

**Why this priority**: Gemini API は従量課金のため、不特定多数がアクセスできる状態は課金リスクがある。

**Independent Test**: 誤ったパスワードでアクセスしても OCR 画面が表示されないことを確認する。

**Acceptance Scenarios**:

1. **Given** 未ログインのユーザーがアプリを開く、**When** アクセスする、**Then** ログインフォームのみが表示され OCR 機能は利用できない
2. **Given** ログインフォームが表示されている、**When** 正しいユーザー名・パスワードを入力する、**Then** OCR 機能にアクセスできる
3. **Given** ログインフォームが表示されている、**When** 誤ったユーザー名・パスワードを入力する、**Then** エラーメッセージが表示され OCR 機能にはアクセスできない
4. **Given** ログイン済みのユーザーが操作している、**When** ログアウトボタンを押す、**Then** ログインフォームに戻る

---

### User Story 3 - OCR に使うモデルを選択する (Priority: P3)

ユーザーは OCR 処理に使用する AI モデルを選択できる。各モデルには精度の目安が表示され、用途に応じて選択できる。

**Why this priority**: 精度とコスト・速度のトレードオフをユーザーが判断できるようにするため。

**Independent Test**: モデル選択 UI が表示され、選択したモデルで OCR が実行されることを確認する。

**Acceptance Scenarios**:

1. **Given** ユーザーがログイン済み、**When** アプリを開く、**Then** モデル選択 UI と各モデルの精度目安が表示される
2. **Given** モデルが選択されている、**When** 撮影して OCR を実行する、**Then** 選択したモデルで処理される

---

### User Story 4 - 抽出テキストをコピーする (Priority: P4)

ユーザーは抽出されたテキスト結果をクリップボードにコピーして他のアプリで利用できる。

**Why this priority**: 認識結果をそのまま活用できるようにすることで、アプリの実用性が高まる。

**Independent Test**: テキストが表示された後、コピーボタンを押してクリップボードに内容が入ることを確認する。

**Acceptance Scenarios**:

1. **Given** OCR 結果テキストが表示されている、**When** コピーボタンを押す、**Then** テキストがクリップボードにコピーされる

---

### Edge Cases

- 文字が含まれていない画像（風景写真など）を撮影した場合、「文字が検出されませんでした」などの適切なメッセージを表示する
- 極端に低解像度・高解像度の画像を撮影した場合でも、エラーなく処理を試みる
- 処理に時間がかかる場合、ユーザーが待機中であることを認識できるフィードバックを提供する
- 外部サービスへの接続エラーが発生した場合、ユーザーにわかりやすいエラーメッセージを表示する
- カメラへのアクセス権限が拒否された場合、権限設定の案内を表示する
- カメラを搭載していないデバイスでは、カメラが利用できない旨のメッセージを表示する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムはユーザーがブラウザからカメラを起動して撮影できること
- **FR-002**: システムは撮影した画像から文字テキストを抽出し、ユーザーに表示すること
- **FR-009**: システムは認証済みユーザーのみ OCR 機能を利用できるよう、ログインフォームでアクセスを保護すること
- **FR-012**: システムはユーザーが OCR モデルを選択できる UI を提供すること
- **FR-013**: システムは各モデルの精度目安をモデル選択 UI 上に表示すること
- **FR-010**: 認証情報（ユーザー名・パスワード）はアプリの設定として管理し、ソースコードに含めないこと
- **FR-011**: ログイン済みユーザーがログアウトできること
- **FR-005**: システムは処理中の状態をユーザーに視覚的に伝えること
- **FR-006**: システムは抽出されたテキストをユーザーがコピーできる手段を提供すること
- **FR-008**: システムは文字が検出されない場合・エラー発生時に適切なメッセージを表示すること

### Key Entities

- **Image（画像）**: ユーザーがカメラで撮影する入力データ
- **OCR Result（OCR 結果）**: 画像から抽出されたテキスト文字列

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーはカメラで撮影してから OCR 結果を受け取るまでの操作を 1 分以内に完了できる
- **SC-002**: 明瞭な印刷文字を含む標準的な画像に対して、認識結果が目視で判読可能な品質であること
- **SC-003**: ユーザーは操作手順を説明されることなく、直感的にアプリを使用できること
- **SC-004**: エラー発生時やカメラ非対応時に、ユーザーが次の行動を理解できるメッセージが表示されること

## Assumptions

- 利用者はブラウザからアクセスするエンドユーザーを想定し、技術的知識は不要
- 入力はカメラ撮影のみとする（ファイルアップロードは v1 スコープ外）
- カメラ機能はブラウザのカメラアクセスを前提とする（アクセス権限が必要）
- 認証はユーザー名・パスワードによるシンプルなログインフォームとする（OAuth・SSO は v1 スコープ外）
- 認証情報は Streamlit secrets で管理し、GitHub リポジトリには含めない
- バッチ処理（複数画像の一括処理）は v1 スコープ外
- 認識結果の保存・履歴管理は v1 スコープ外
- インターネット接続が利用可能な環境を前提とする（外部 AI サービスを使用するため）
- デプロイ先は Streamlit Community Cloud とする
